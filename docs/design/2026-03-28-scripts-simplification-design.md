# Design: ACTS Scripts Simplification with Human-in-the-Loop Gates

**Date:** 2026-03-28
**Topic:** Replacing scripts with agent-executed operations + approval gates
**Version:** ACTS v0.3.0

---

## Problem Statement

The current ACTS implementation has redundant layers:

1. **Scripts** (`acts.sh`) provide human-visible status checks
2. **Operations** (`.acts/operations/*.md`) contain the full workflow steps
3. **Skills** expose operations to agents

This means `preflight` runs twice — once as a script for the human, once as an operation for the agent. The script version is strictly worse (no context ingestion, no state updates).

**Goal:** Simplify to a single agent-executed layer while preserving human oversight.

---

## Solution: Report + Gate Pattern

Replace script execution with agent-presented reports and explicit approval gates.

### Report Protocol

Standard formats defined in `.acts/report-protocol.md`:

| Report | Used in | Purpose |
|---|---|---|
| **Story Board** | preflight, handoff, story-review | Task status overview |
| **Ownership Map** | preflight, handoff | File ownership by completed tasks |
| **Scope Declaration** | preflight | What agent will/won't do |
| **Session State** | session-summary | Build, test, lint, git status |

### Gate Types

| Type | Behavior | Example |
|---|---|---|
| `GATE: approve` | Wait for "yes", abort on "no" | Proceed with T1? |
| `GATE: acknowledge` | Show report, any response continues | Plan created |
| `GATE: reject` | Continue on silence, abort on "no" | Ready to commit? |

---

## Updated Operations

Each operation gets GATE steps at key decision points:

| Operation | Gate Location | Type | Presented |
|---|---|---|---|
| `preflight` | After context ingestion | `approve` | Story Board + Ownership Map + Scope |
| `task-start` | Before first commit | `acknowledge` | Scope reconfirmation |
| `session-summary` | After writing, before commit | `approve` | Session State |
| `handoff` | After briefing | `approve` | Handoff briefing |
| `story-review` | After acceptance check | `approve` | AC checklist + PR description |
| `story-init` | After plan creation | `acknowledge` | Spec + plan summary |

---

## Script Fate

| Script | Current | Fate | Rationale |
|---|---|---|---|
| `acts.sh` | Main CLI | **Remove** | Redundant with operations |
| `acts.sh status` | Task board | **Remove** | Agent presents Story Board in gates |
| `acts.sh preflight` | Validation display | **Remove** | Operation + gate replaces it |
| `acts.sh worktree` | Git worktree | **Remove** | Agent runs `git worktree add` |
| `acts.sh compress` | Extract sections | **Remove** | Operation reads files directly |
| `acts.sh context-budget` | Show budgets | **Remove** | Agent reads frontmatter |
| `acts.sh ops-meta` | List operations | **Remove** | Agent reads directory |
| `acts.sh adapter` | Generate config | **Remove** | Agent generates on demand |
| `compress-sessions.sh` | Extract data | **Remove** | Operation handles extraction |
| Adapter scripts (5) | Tool configs | **Remove** | Agent generates on demand |
| `validate.sh` | Conformance | **Keep** | CI/CD only, no agent in pipelines |

**Remaining scripts:**
```text
scripts/
└── validate.sh    # CI/CD deterministic validation only
```

## Flow Comparison

### Before (3-layer)
```
15:00  Human: ./scripts/acts.sh preflight T1 alice
15:01  Human: reads output, decides OK
15:02  Human: "Load skill acts-preflight for T1"
15:03  Agent: reads operation, does ingestion
15:05  Agent: updates state, ready
```

### After (1-layer)
```
15:00  Human: "Start working on T1"
15:01  Agent: loads skill, reads operation
15:03  Agent: does context ingestion
15:05  Agent: presents Story Board + Scope + GATE: approve
       ┌─────────────────────────────────────┐
       │ PROJ-305 — Add Two-Factor Auth      │
       │ T1: TODO → I will build TOTP setup  │
       │                                     │
       │ Ready to proceed? (yes/no)          │
       └─────────────────────────────────────┘
15:06  Human: "yes"
15:06  Agent: updates state, ready
```

**Fewer steps. No script knowledge. Human still approves.**

## Gate Implementation

In operation files:

```markdown
## Steps

6. **CONTEXT INGESTION**
   Follow Context Protocol. Then present Reports.

   ### Report
   Present Story Board per .acts/report-protocol.md
   Present Ownership Map showing completed task files
   Present Scope Declaration for this task

   ### Gate
   GATE: approve
   "Ready to proceed with <task_id>? (yes/no)"
   Do NOT update state.json until developer confirms.
```

## CI/CD Preservation

`validate.sh` stays for headless pipelines:

```yaml
# .github/workflows/acts.yml
- run: ./scripts/validate.sh --json
```

For interactive validation, add optional `validate` operation:
```
User: "Run ACTS validation"
Agent: loads acts-validate, presents results
```

## Approval

This design approved via brainstorming session 2026-03-28.

**Key decisions:**
- Approach C: Inline gates + shared report protocol
- Remove 9 scripts, keep only `validate.sh`
- 4 standard report formats
- 3 gate types (approve/acknowledge/reject)
