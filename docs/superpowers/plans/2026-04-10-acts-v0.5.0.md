- [ ] **Step 1: Update Section 7.2 Worktree Model with specific guidance for code review tools**

```markdown
### 7.2 Worktree Model

For multi-developer stories, each story MUST live in a dedicated git worktree:

```text
<repo-root>/
├── (main working directory)        # main branch
└── ../worktrees/
    └── <STORY_ID>/                 # story/<STORY_ID> branch
        ├── .story/
        ├── .acts/
        └── <application code>
```

**Important Notes for Code Review Tools:**
When using code review tools like GitHuman in worktrees:
1. The tool MUST be run from within the worktree directory
2. Ensure the tool has access to the full git history (may need to fetch)
3. If the tool requires access to the main repository, configure it to use the worktree's git directory
4. Always verify the tool is operating on the correct branch before submitting reviews
```
